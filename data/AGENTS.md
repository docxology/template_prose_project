# Data Directory Instructions

Keep this directory limited to small, versioned project inputs that support the
template prose exemplar.

- Edit `claim_ledger.yaml` only when manuscript claims, prose metrics, or
  source-backed numeric facts change.
- Keep generated reports and figures out of this directory; pipeline outputs
  belong under `output/`.
- Preserve the evidence-registry schema fields (`claim_id`, `kind`, `value`,
  `source`, `source_tier`, `freshness`, `artifact_path`) when adding claims.
